Airport systems


Airport systems are a collection of integrated technologies and processes designed to streamline and manage the complex operations of an airport. These systems cover a wide range of functions, from flight management and passenger processing to security and maintenance.




Airport Operations Database (AODB)


The Airport Operations Database (AODB) is a comprehensive system designed to manage and streamline airport operations. It serves as the central hub for storing and tracking vital flight, maintenance, and operational information. By managing activities across both landside and airside operations, AODB ensures the seamless flow of airport processes and provides accurate financial data for billing purposes.


Key Features and Modules:

  • Seasonal Schedule: The AODB system efficiently manages and updates seasonal flight schedules, ensuring that all flights are accurately recorded and tracked according to seasonal changes and demand.
  • Flight Operations:This module tracks and manages all aspects of flight operations, including arrival and departure details, gate assignments, and flight status updates. It enhances operational efficiency and ensures timely handling of all flights.
  • Airport Data Management: AODB consolidates all relevant data related to airport operations, including flight information, maintenance schedules, resource allocations, and more. This centralization improves decision-making and coordination across various airport departments.
  • Business Rules Configuration: This feature enables airports to configure and apply business rules specific to their operational needs. Customizable rules ensure that airport processes, such as billing, flight management, and customer services, align with the airportâ€™s operational standards.
  • AODB Administration: The administrative module of AODB allows for system configuration and management, user access control, and system maintenance. It ensures that the database operates smoothly and meets the evolving needs of airport management.




RMS (Resource Management System)


RMS (Resource Management System) is designed to manage and allocate critical resources within an airport system, such as stands, gates, carousels, and check-in counters. It ensures optimal usage and scheduling of the resources through various features of which are given below


Key Features and Modules:

  • Resource Configuration Setting up and defining available resources.
  • Resource Allocation Requirements:Defining the conditions for allocating resources to specific tasks.
  • Resource Rule Configuration: Creating rules for managing resource usage and allocation.
  • Resource Allocation: Assigning resources to tasks based on availability and requirements.
  • RMS Administration: Overseeing and managing the overall RMS operations.
  • Billing: Managing the financial aspects, including the billing of resources based on usage.




FIDS (Flight Information Display System)


The Flight Information Display System (FIDS) enables authorized users to design and publish screen layouts for display devices. It manages images and videos shown on information and advertising screens, communicating with the Display-Manager to receive templates, static data, and dynamic updates. The user interface is designed for ease of use, allowing adjustments such as modifying roll-on and roll-off times and creating custom FIDS templates.


Types of FIDS Displays:

  • Check-In Counter Activation
  • Gate Activation
  • Baggage Reclaim
  • Emergency Information
  • Lane Information Displays (LIDS) (e.g., Nationals, Tourist Visa, Diplomatic/Crew, etc.)
  • Staff FIDS (Accessible through a browser-based client on the Staff Intranet)
  • Public FIDS Screens (Display at least 3 hours of flight information prior to the scheduled time)

Interfaced Systems:

  • AODB (Airport Operational Database): Integrated through an independent information broker. In case of prolonged interface failure, a manual input option is available through the FIDS Admin client to keep displays updated with changes in the AODB.
  • Digital Media System (DMS): DMS content is displayed on FIDS screens by defining a field within the FIDS page template to play media content from the IPTV server. The FIDS DDC output is directed to a DMS encoder, which streams the content as a multicast channel received by DMS set-top boxes.




BRS (Baggage Reconciliation System)


The Baggage Reconciliation System (BRS) is used by ground handlers in the baggage sorting area to ensure accurate baggage tracking and loading. It operates through workstations, printers, and mobile devices connected over a wireless network. Airlines can also access the system from their offices to monitor baggage operations for their flights.



System Components


1. Application/Database Server
The core of the system, responsible for:
  • Storing information on bags, flights, passengers, and users
  • Sending and receiving baggage information from airline Departure Control Systems (DCS)
  • Receiving flight details from the AODB
  • Providing real-time baggage data to workstations and handheld terminals (HHTs)

2. Hand-Held Terminals (HHTs)
Used by baggage handlers to scan bags as they are loaded into containers, carts, or aircraft. HHTs provide real-time data on baggage status and communicate with the server to ensure correct loading.

3. Workstations
Ground handlers use workstations for:
  • Managing the baggage reconciliation process
  • Controlling the allocation and return of HHTs
  • Printing container sheets and reports
  • Checking baggage status
  • Matching passengers with their bags, including lost and found items
  • Updating flight, user, and equipment attributes

4. Printers
  • LaserJet Printers: Used for printing manifests, baggage lists, and container sheets
  • Bag Tag Printers: Used for printing crew bag tags, re-flighted bag tags, and reprints

5. Message Distribution System (MDS)
All airlines' Departure Control Systems (DCS) are connected to a centralized messaging hub for exchanging baggage-related messages:
  • BSM (Baggage Source Message)
  • BTM (Baggage Transfer Message)
  • BPM (Baggage Processed Message)s
  • BUM (Baggage Unload Message)

Main Functions

  • Integration with DCS: Receives and processes baggage handling messages from airline systems
  • Handheld Terminal Functions: Scanning and tracking baggage loading/unloading and Offloading bags if a passenger cancels their trip after check-in.
  • Apron Mode Functions: Associates bag tags with aircraft holds, Associates containers/trolleys with aircraft holds, Displays error messages if a bag or container is loaded in the wrong location and Allows selection of bags to be offloaded.
  • User Application Functions (Administration Console): Configuring containers and trolleys for each flight, Generating and printing reports, Printing container/trolley tags, Baggage message inquiries and System parameter and archive configuration.
  • Baggage Information Display System: Displays real-time baggage information for loaders in the makeup area to improve efficiency and accuracy.




CUPPS (Common Use Passenger Processing System)


CUPPS provides a standardized platform for handling airline passenger processing across various airport locations. It supports multiple peripherals and integrates with key airport systems to facilitate check-in, boarding, baggage handling, and passenger management.


Supported Peripherals

  • Boarding Pass Printers (ATB): Prints boarding passes for passengers.
  • Bag Tag Printers (BTP): Prints baggage tags for checked luggage.
  • Boarding Gate Readers (BGR): Scans boarding passes at the gate for passenger boarding.
  • Multi-Document Reader Keyboards (MSR/OCR): Reads passports, visas, and other travel documents.
  • Barcode Readers (LSR): Scans barcodes on boarding passes and baggage tags.

Main Locations

  • Check-in Counters: Passenger check-in and baggage drop.
  • Boarding Gates: Passenger boarding and departure processing.
  • Lost & Found: Managing lost baggage and passenger claims.
  • Transfers: Assisting passengers with connecting flights.
  • Back Offices: Administrative and support functions.

Passenger Handling & Common Language Facility (CLF)

This system enables airport handling agents and airlines to interact with multiple airline systems through a unified graphical interface. Key functionalities include:
  • Passenger check-in processing.
  • Displaying and editing passenger information.
  • Managing passenger seat assignments and changes.
  • Printing tickets, boarding passes, and baggage tags.
  • Passenger boarding using gate readers.
  • Accessing flight information, seat inventory, and seat map analysis.

Departure Control System (DCS)

DCS automates the passenger check-in and boarding process, providing a fast and scalable solution that supports:
  • Flight schedules, aircraft inventory, and seat plans.
  • Passenger weight and baggage weight management.
  • Message exchanges with baggage handling systems.

Weight & Balance System

The Weight & Balance module ensures safe aircraft loading and dispatching, including compliance with legal documentation requirements.

Operational Workflow

  • Workstation is powered on and booted.
  • User logs in with credentials.
  • System loads user desktop, security settings, and authorized applications.
  • User selects the required application from the menu.
  • User accesses the Departure Control System (DCS) and begins processing passengers.

System Interfaces

  • Baggage Reconciliation System (BRS) : Enables ground handlers and airlines to track baggage.
  • Bag Message System (BMS): Sends baggage messages when tags are printed at check-in.
  • Airport Operational Database (AODB): Allows agents to update flight display screens and nsures efficient check-in counter allocation.
  • Passenger & Security Data Systems: Transmits passenger lists and passport details to authorities.
